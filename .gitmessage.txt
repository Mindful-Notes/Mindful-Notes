# ============================================
# Git Commit Message Template (Gitmoji)

# 형식:
# <gitmoji> <type>: <subject>

# 커밋 메시지는 아래 형식으로 통일!
# <이모지> <type>: <제목>

# 예시:
#  1. ✨ Feat: 기능 추가
# ✨ feat: 회원가입 API 추가
# ✨ feat: 로그인 API 및 JWT 발급 구현
# ✨ feat: 일기 작성/조회/수정/삭제 CRUD 구현
# ✨ feat: 명언 랜덤 조회 API 추가
# ✨ feat: 자기성찰 질문 랜덤 조회 API 추가

#  2. 🐛 Fix: 버그 수정
# 🐛 fix: 토큰 만료 처리 오류 수정
# 🐛 fix: 일기 검색 조건 누락 문제 수정
# 🐛 fix: 중복 회원가입 예외 처리 보완

#  3. 💡 chore: 잡일 / 기능추가 없는 변경
# 💡 chore: 프로젝트 기본 폴더 구조 생성
# 💡 chore: ruff 설정 및 코드 스타일 정리
# 💡 chore: 환경 변수 파일(.env) 예시 추가
# 💡 chore: 커밋 메시지 템플릿 추가

#  4. 📝 docs: 문서 수정
# 📝 docs: 프로젝트 실행 방법 README 추가
# 📝 docs: API 명세 문서 업데이트
# 📝 docs: 환경 변수 설정 가이드 작성

#  5. 🚚 build: 빌드/배포 관련
# 🚚 build: pyproject.toml 의존성 추가 및 정리
# 🚚 build: 배포용 설정 파일 추가

#  6. ✅ test: 테스트 관련
# ✅ test: 회원가입/로그인 API 테스트 추가
# ✅ test: 일기 CRUD 테스트 코드 작성

#  7. ♻️ refactor: 리팩토링
# ♻️ refactor: auth 로직을 service 레이어로 분리

# - 제목은 영문 기준 50자 이내로
# - 제목 첫글자를 대문자로
# - 제목 끝에 . 금지
# - 제목은 명령조로
# - 본문은 영문 기준 72자마다 줄 바꾸기
# - 본문은 어떻게보다 무엇을, 왜에 맞춰 작성하기
